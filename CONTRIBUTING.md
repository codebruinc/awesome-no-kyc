# Contributing to Awesome No-KYC

Thank you for your interest in contributing! This document provides guidelines for contributing to this list.

## Criteria for Inclusion

To be added to this list, a service must meet **most** of these criteria:

| Criteria | Required? | Description |
|----------|-----------|-------------|
| No Government ID | **Yes** | Must not require passport, driver's license, or national ID |
| Minimal Data Collection | **Yes** | Only collects what's technically necessary |
| Cryptocurrency Accepted | Preferred | Accepts BTC, XMR, Lightning, or other crypto |
| No Account Required | Preferred | Can be used without registration, or registration is minimal |
| No Email Required | Preferred | Accepts disposable emails or no email needed |
| No Phone Verification | Preferred | No SMS verification required |
| Good Reputation | **Yes** | No known scams, exit scams, or major security issues |
| Currently Operational | **Yes** | Service is active and working |

### Updating Existing Entries

If a service has changed its policies:

1. **Verify** the change on the service's website
2. **Update** the relevant entry
3. **Submit** a PR explaining the change

### Removing Services

Services should be removed if they:

- Now require KYC/ID verification
- Have exit scammed or are confirmed scams
- Are no longer operational
- Have major unresolved security issues

Submit a PR with evidence of the issue.

## Reporting Issues

If you find:

- A dead link
- A service that now requires KYC
- Incorrect information
- A scam or security issue

Please [open an issue](https://github.com/codebruinc/awesome-no-kyc/issues) with details.

## Code of Conduct

- Be respectful and constructive
- Focus on privacy and user benefit
- No promotion of illegal activities
- No spam or self-promotion (legitimate services welcome)

## Questions?

Open an issue or reach out to the maintainers.
